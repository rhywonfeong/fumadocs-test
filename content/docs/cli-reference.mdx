## 用法

```bash
npx -y hpsetup@latest <key> [product] [options]
HEROUI_KEY=<key> npx -y hpsetup@latest [product] [options]
```

## 位置参数

### `<key>`

API Key，以 `hp_` 开头。也可以通过 `HEROUI_KEY` 环境变量传入。

### `[product]`

可选。指定要安装的产品：

| 值 | 产品 |
|----|------|
| `react` | HeroUI React Pro（默认） |
| `native` | HeroUI Native Pro |

不指定时进入交互式选择。

## 选项

| 选项 | 说明 |
|------|------|
| `--update` | 强制更新到最新版本 |
| `--auto` | 跳过所有交互提示，从 `package.json` 自动检测配置 |
| `--dry-run` | 仅显示将要执行的操作，不做实际更改 |
| `--no-cache` | 绕过所有缓存，从源站拉取 |
| `-h, --help` | 显示帮助信息 |

## CI 模式

当检测到 `CI` 环境变量时，自动启用以下行为（无需手动传参）：

1. **`--update` 隐含启用** — 始终安装最新版本
2. **`interactive = false`** — 跳过所有交互提示

这意味着在 CI 环境中只需：

```bash
HEROUI_KEY=hp_xxxx npx -y hpsetup@latest
```

等价于手动添加了 `--update` 标志。

## 版本检查

hpsetup 通过 npm registry 检查最新版本，使用递增超时重试策略（3s → 6s → 10s），共 3 次。

在 `--update` 模式下，如果版本检查全部失败，包会被标记为"版本未知"并强制走更新流程，确保不会因为网络波动而跳过更新。

## 下载容错

CDN 下载失败时的 fallback 链：

```
CDN 下载失败 → 尝试当前版本本地缓存 → 尝试上一版本本地缓存 → 退出
```

在 `--update` 模式下会保存 `previousVersion`，用于在更新失败时回退到已知可用的版本。

## Expo SDK 兼容

安装 Native Pro 时，hpsetup 会根据项目中的 Expo SDK 版本自动映射 peer 依赖版本：

| Expo SDK | `@shopify/react-native-skia` |
|----------|------------------------------|
| 52 | 1.5.0 |
| 53 | 2.0.0-next.4 |
| 54 | 2.2.12 |
| 55 | 2.4.18 |

这些版本通过 `npx expo install` 批量测试确认兼容。
