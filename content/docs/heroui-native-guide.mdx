克隆 [heroui-native-example](https://github.com/heroui-inc/heroui-native-example) 到本地，然后安装依赖并启动。

<Steps>
<Step>

### 克隆项目

<Tabs items={['p', 'git']} defaultValue="p">
  <Tab value="p">

确保已安装 [p cli](https://github.com/ru-yaka/p)：

```bash
bun install -g ru-yaka/p
```

然后克隆项目，它会自动放到 `~/.p/projects/` 下并用 IDE（默认 cursor，`p config` 可配）打开：

```bash
p clone heroui-inc/heroui-native-example
```

![p clone](https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260510190300680.png)

  </Tab>
  <Tab value="git">

```bash
git clone https://github.com/heroui-inc/heroui-native-example
```

  </Tab>
</Tabs>

</Step>
<Step>

### 安装依赖

<Tabs groupId="pkg" items={['pnpm', 'bun', 'npm']} defaultValue="pnpm">
  <Tab value="pnpm">

```bash
pnpm install
```

  </Tab>
  <Tab value="bun">

```bash
bun install
```

  </Tab>
  <Tab value="npm">

```bash
npm install
```

  </Tab>
</Tabs>

<Callout type="info">
  不推荐使用 yarn。也可以删除 `package-lock.json`，改用其他包管理器安装。
</Callout>

</Step>
<Step>

### 允许构建脚本

安装完成后运行 start，pnpm 可能会阻止 `unrs-resolver` 的构建脚本：

![unrs-resolver 报错](https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260510113307101.png)

运行 `pnpm approve-builds` 交互式允许（空格选中后回车），也可以直接编辑 `pnpm-workspace.yaml` 设为 true。

<Callout type="info">
unrs-resolver 是 Rust 编写的高性能模块解析器，需要 postinstall 脚本编译对应平台的二进制文件。
</Callout>

</Step>
<Step>

### 启动项目

<Tabs groupId="pkg" items={['pnpm', 'bun', 'npm']}>
  <Tab value="pnpm">

```bash
pnpm run start
```

  </Tab>
  <Tab value="bun">

```bash
bun run start
```

  </Tab>
  <Tab value="npm">

```bash
npm run start
```

  </Tab>
</Tabs>

![启动项目](https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260509232236630.png)

发现有两个依赖不兼容当前的 expo 版本（测试时为 55）。

</Step>
<Step>

### 修复依赖

停止 start，自动安装当前 expo 兼容版本：

```bash
npx expo install react-native react-native-worklets
```

![修复依赖](https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260509232543541.png)

</Step>
<Step>

### 验证运行

重新 start，然后用 Expo Go 55 扫码测试。

<Callout type="warn">
  Expo Go 版本必须和当前 expo SDK 版本匹配，否则无法运行。
</Callout>

一切正常：

<img
  src="https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260509233459361.png"
  alt="运行成功"
  width="320"
/>

</Step>
</Steps>

## 添加为模板

添加当前项目为模板，下次可以从本地快速新建项目（已修复依赖）：

<Tabs items={['tp', 'template', 'templates']} groupId="p-tp">
  <Tab value="tp">

```bash
p tp add . heroui-native
```

  </Tab>
  <Tab value="template">

```bash
p template add . heroui-native
```

  </Tab>
  <Tab value="templates">

```bash
p templates add . heroui-native
```

  </Tab>
</Tabs>

![p tp add](https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260510191342462.png)

## 从模板新建项目

<Tabs items={['new', 'n', 'create']} groupId="p-new">
  <Tab value="new">

```bash
p new my-heroui-native-app -t heroui-native
```

  </Tab>
  <Tab value="n">

```bash
p n my-heroui-native-app -t heroui-native
```

  </Tab>
  <Tab value="create">

```bash
p create my-heroui-native-app -t heroui-native
```

  </Tab>
</Tabs>

![p new](https://image-bed-1315938829.cos.ap-nanjing.myqcloud.com/new/image-20260510191435746.png)

<Callout type="info">
  `p new <name>` 可以快速创建项目（放在 `~/.p/projects/` 目录下统一管理），并用配置 IDE 打开（默认 cursor，可 `p config` 配置）。忘记模板名可以 `-t` 留空交互式选择，或 `-t <fuzzy_name>` 模糊搜索。
</Callout>
